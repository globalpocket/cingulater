# Brownie Architecture V2: Platform-Agnostic Engine

## 1. 概要
Brownie V2 は、コアロジックを外部プラットフォーム（GitHub 等）から完全に分離した、主権型・汎用推論エンジンです。すべてのインフラ操作は MCP (Model Context Protocol) を通じて実行されます。

## 3. 主要コンポーネント

### Core 層 (思考・制御)
- **Orchestrator**: プロジェクトのライフサイクルと状態遷移を管理。プラットフォーム固有の仕様（GitHub API 等）を知らず、汎用的な要求を出します。
- **Infrastructure Bridge**: Core からの要求（例：通知の取得）を受け、適切な MCP サーバーへディスパッチする絶縁体。

### Infrastructure 層 (実務・実行)
- **GitHub Platform MCP**: GitHub との API 通信を一手に引き受ける。
- **Git MCP**: ローカル環境での Git ファイル操作を担当。
- **Task Reasoning MCP**: LLM を用いた推論ループ処理の実行本体。

## 4. アーキテクチャのメリット
1. **GitHub からの独立**: GitHub がダウンしても、あるいは他の Git ホスティング（GitLab等）に切り替えても、Core を変更せずに対応可能。
2. **インターフェースの統一**: CLI や Slack など、異なる UI から同じ推論ロジックを再利用可能。
3. **セキュリティ**: ホスト環境やリポジトリへの直接アクセスを MCP レベルで制限・管理可能。
